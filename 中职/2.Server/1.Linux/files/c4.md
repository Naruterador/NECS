## 题目
- 在T1目录下创建f1.json文件，内容为/etc/sysconfig/下包含所有子目录中的文件名及目录名的层次结构信息，使用JSON格式表示，例如：{"type":"file","name":"anaconda"}、{"type":"directory","name":"/etc/sysconfig/","contents":[]}。

## 答案
```shell
dnf install jq

find /etc/sysconfig/ -print | awk '{if($0 ~ /\/$/) print "{\"type\":\"directory\",\"name\":\"" $0 "\"}"; else print "{\"type\":\"file\",\"name\":\"" $0 "\"}"}' > /tmp/skills_tmp/T1/f1.json

#指令解释
1. find /etc/sysconfig/ -print
find 命令: 用于递归地查找指定目录中的文件和子目录。
/etc/sysconfig/: 指定查找的根目录。
-print: 输出每个找到的文件或目录的路径。
效果:列出 /etc/sysconfig/ 目录下的所有文件和子目录的路径。

2. |
管道符（|）: 将 find 命令的输出传递给 awk 命令处理。

3. awk '{...}'
awk 是一个强大的文本处理工具，它逐行处理输入数据。
{if($0 ~ /\/$/) ... else ...}: awk 中的 if 语句判断当前行的内容（即文件路径）是否以 / 结尾，用于区分文件和目录：
$0：表示当前行的内容（即每个文件或目录的路径）。
~ /\/$/: 检查路径是否以 / 结尾。正则表达式 /\/$/ 表示字符串结尾是 /，这意味着当前路径是一个目录。
print "{\"type\":\"directory\",\"name\":\"" $0 "\"}": 如果路径是目录（以 / 结尾），则输出一个包含 "type": "directory" 和 "name": "<目录路径>" 的 JSON 格式对象。
else print "{\"type\":\"file\",\"name\":\"" $0 "\"}": 如果路径不是目录（即是文件），则输出一个包含 "type": "file" 和 "name": "<文件路径>" 的 JSON 格式对象。
效果：将每个文件或目录的路径转换为 JSON 对象，表示该路径是文件还是目录，并添加对应的 "type" 和 "name" 字段。

4. > /tmp/skills_tmp/T1/f1.json
>: 重定向符号，用来将标准输出保存到文件中。

/tmp/skills_tmp/T1/f1.json: 指定输出文件的路径。最终的 JSON 内容将写入到这个文件。
效果: 将 awk 输出的 JSON 数据保存到 /tmp/skills_tmp/T1/f1.json 文件中。

```
